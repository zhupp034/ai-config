# Task Record Guidance

Use this guidance when maintaining a task record during implementation.

## Language
- Task records and delivery evidence default to Chinese.
- Commands, paths, code ids, and commit types stay in English.

## Minimum sections
- Basic info
- Plan
- Plan review
- Implementation
- Code review
- Test case design
- Test case review
- Test execution
- Bugfix and retest
- Final report

## Good practice
- Update the task record before and after meaningful changes.
- Keep evidence short but concrete.
- Prefer command names, affected modules, and decision notes over vague summaries.
- Link a bugfix to the failed scenario that triggered the retest.

## Weak patterns to avoid
- Writing the whole record after implementation finishes.
- Keeping review comments only in chat.
- Saying "tested" without commands, steps, or result.
